Frequently Asked Questions

How do I run the install script?

Open PowerShell as Administrator, then run:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

.\install-windows.ps1

winget handles all dependencies (Git, Docker Desktop, Ollama) automatically.

Will Ollama use my GPU?

Ollama auto-detects NVIDIA GPUs (requires CUDA drivers) and AMD GPUs (requires ROCm). Install the latest drivers from your GPU vendor, then restart Ollama. Check with ollama run llama3.2:3b and look for "GPU" in the output.

Can I use Claude, GPT-4, Gemini, or Grok?

Yes. Open ThreadWeaver → Settings → paste your API key for the provider you want. You can switch providers mid-conversation.

Why are tools off by default for local models?

Sending 28 tool schemas on every request slows down small local models. With tools off, local chat is fast. Turn them on in Settings when you need the model to check sensors, write files, or control things. External providers (Claude, GPT-4) handle tools natively.

How do I add more models?

From PowerShell: ollama pull <model-name>. Browse ollama.com/library for available models. Machines with 16GB+ RAM can run larger models like llama3.1:70b.

Is my data private?

With the local provider, everything stays on your machine. Model weights are stored by Ollama in %USERPROFILE%\.ollama\models. Conversations are in the ThreadWeaver Docker volume. Nothing leaves your machine unless you switch to an external provider.

How do I stop or restart everything?

Stop containers: cd $env:USERPROFILE\picocluster-claw; docker compose -f docker-compose.yml -f docker-compose.windows.yml down

Start containers: docker compose -f docker-compose.yml -f docker-compose.windows.yml up -d threadweaver openclaw portal

Restart Ollama: Close the Ollama tray icon, then reopen it from the Start menu.
